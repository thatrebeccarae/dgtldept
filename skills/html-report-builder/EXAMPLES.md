# HTML Report Builder — Examples

## Example 1: Marketing Audit Report

**Prompt:**
> Create an audit report for an email marketing program. Here are the key metrics: open rate 18.2% (benchmark 21%), click rate 1.4% (benchmark 2.6%), revenue per recipient $0.12 (benchmark $0.31), list growth rate -2.1%.

**Expected output:** A standalone HTML report with:
- Dark cover page with report title, date, and scope
- Executive summary with 4 KPI cards (open rate, click rate, RPR, list growth) color-coded against benchmarks
- Findings sections: List Health, Engagement Trends, Revenue Attribution, Deliverability
- Each section has a data table or chart + callout box with the bottom-line insight
- Recommendation cards: prioritized quick wins, strategic initiatives, maintenance items
- Appendix with raw data tables

## Example 2: Competitive Analysis

**Prompt:**
> Build a competitive analysis report comparing our product against 3 competitors across pricing, features, market positioning, and customer sentiment.

**Expected output:** A report with:
- Cover page
- Executive summary with positioning callout
- Feature comparison data table with severity coloring (green = advantage, red = gap, amber = parity)
- Pricing comparison section with KPI cards
- Market positioning analysis with callout boxes
- Customer sentiment section with scored metrics
- Strategic recommendations

## Example 3: Quarterly Performance Review

**Prompt:**
> Create a Q4 performance report. Revenue: $2.4M (target $2.1M). New customers: 847 (target 600). Churn: 4.2% (target < 3%). CSAT: 4.1/5. Include channel breakdown: Organic 42%, Paid 31%, Email 18%, Referral 9%.

**Expected output:** A report with:
- Cover page with quarter and date range
- Executive summary with 4 KPI cards, 3 green (above target), 1 red (churn above target)
- Channel performance section with data table showing each channel's contribution
- Goal attainment section with progress indicators
- Priority banner flagging churn as critical
- Recommendations focused on churn reduction strategies
- Appendix with monthly breakdown tables

## Example 4: Technical Assessment

**Prompt:**
> Create a report assessing a client's analytics tracking setup. Findings: 12 of 18 required events are tracked, 3 have incorrect parameters, page load time is 4.2s (target < 2s), no server-side tracking configured.

**Expected output:** A report with:
- Cover page
- Executive summary with a checklist-style scorecard (tracked/missing/broken events)
- Event inventory data table with status coloring per event
- Performance section with KPI cards (page load, coverage %)
- Gap analysis with priority banners for critical missing events
- Implementation roadmap as numbered recommendation cards with effort/impact badges
